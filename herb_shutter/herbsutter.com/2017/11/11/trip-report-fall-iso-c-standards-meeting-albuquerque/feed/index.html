]]> By: Casey Carter https://herbsutter.com/2017/11/11/trip-report-fall-iso-c-standards-meeting-albuquerque/#comment-40026 Sun, 12 Nov 2017 09:09:12 +0000 http://herbsutter.com/?p=3624#comment-40026 In the range-based-for example, “for (auto& x : f().items())” is perfectly valid if items() returns a prvalue or a glvalue whose lifetime is not tied to the lifetime of the return value of f. The UB case is when f returns a prvalue and items returns a reference to or view over a member container of f’s return value. You can argue that the example is dangerous since it’s not clear *what* f returns, but it would more clearly illustrate the problem and the value of the new feature if “f().items()” was something like “vector<vector>{{1,2,3}}[0]”.
