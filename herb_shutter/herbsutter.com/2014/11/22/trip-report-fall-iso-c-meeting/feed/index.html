]]> By: Herb Sutter https://herbsutter.com/2014/11/22/trip-report-fall-iso-c-meeting/#comment-32541 Tue, 25 Nov 2014 18:51:28 +0000 http://herbsutter.com/?p=2541#comment-32541 @Madura A.: I agree we should have a way to split strings, but — surprise — you usually don’t really want to actually physically split strings into multiple string objects. Watch [w]string_view, which is like an ‘iterator range’ on someone else’s string storage… a lot of us are working on that and its cousin array_view. My view is that string_view is almost certain to grow those kinds of functions. And doing it on string_view will be way more efficient than actually physically splitting a string, which would require actually copying characters and most of the time you don’t need to do that, you just want to know where the substrings start and end and read them, which is most efficient when you do it in-place without copying the storage.
